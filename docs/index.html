2 Programming

To to find your way around the code, know following basic objects and structures.

2.1 Input Array

This is the interface to the user, it is a flat array with three fields for each image.

  • Field 1 : Path to the image file
  • Field 2 : A factor for the time-to-show (not yet used)
  • Field 3 : The text to show below the image

How exactly this array makes it's way into the program is not yet clear. There are several possibilities, e.g.

  • Read an XML file via AJAX (as it is done from Daftari).
  • Load a JavaScript file (like the Daftari sitmapdaf.js). This has the great advantage, that it also works for browsing via file protocol. This way is not yet implemented.
  • Use one of the three built-in default image sets. This is done, if no user input is provided otherwise.
  • Use the data-setup attribute of the slider div.

To see an example, have a look at array definition 20190106°1431 for Sldgr.Stt.aFallbackShow_Two in slidegear.js

2.2 Slider object

Imagine the Slider class as one slide projector device. One HTML page can host multiple such devices.

At program start, the HTML page is scanned for div elements of class FurnitureSlideGear. For each found div, one Slider object is created.

The Slider object moves the Magazine array and projects one Slide object after the other. The exact movement and fading is controlled by the Stage object.

2.3 Slide object

On program start, for each image given in the user input array, one Slide object is created, which is stored in the Magazine array.

The Slide object has the following properties:

  • Image Path — The image full filename as read from the input array, but possibly supplemented programmaticaly with some relative path prefix.
  • Image Text — This is the text to print under the each slide, as imported from the input array.
  • Slide Element — This is generated on program start, it is the page HTML div element which holds the image. On program start, all slide elements are set hidden, and on program run, one of them is switched visible.
  • Size factor — This tells how the image is scaled to fit exactly into the Slide. If the original image is too small, it will be blown up, if it is too large, it will be shrinked.
  • Duration Factor — The not yet used number as read from the input array. This is intended to reduce or extend the showtime of individual slides. If this field is empty, the factor is 1.0
  • Fading algorithm — Not yet used. This property might be used to modify the fading style. Actually, the fading style is not a property of one single slide, because there are two or more slides involved in fading.

2.4 Magazine array

The Magazine array, also called Stock array, is the internal working horse. On program start, the Magazine array is filled with the slides, which are created from the input array.

2.5 Stage object

Each Slider is accompanied with one Stage object with properties to control the exact working of the Slider. The Stage object stores the current state of operation and tells the slider what to do in the current iteration.

  • Run Flag bRunning. This tells, whether the Slider is currently running or not.
  • Fade Count iFadeCount. This controls the fading phase, counting the fading steps.
  • Fading Algorithm iFadingAlgo. This tells, what kind of fading shall be applied. (Not yet used)
  • Index Current iNdxCurr. This points into the Magazine to the slide which shall be shown currently. This is the slide to be faded-in.
  • Index Previous iNdxPrev. This points into the Magazine to the slide which was shown previously. This is the slide to be faded-out.
